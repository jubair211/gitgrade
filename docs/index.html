The Problem

Beginners don't see their own patterns — until it's too late.

Health Informatics students are learning to program while managing lectures, clinical concepts, and data systems. Without clear feedback, it is dangerously easy to fall into deadline cramming without noticing — and wonder why skills are not improving. GitGrade reads your git history AND tracks your file activity automatically — giving you the full honest picture of your coding habits, so you can fix them before the semester slips away.


How It Works
Two layers of automatic insight.

No manual input. No forms. Just run two commands.

LAYER 1 — GIT HISTORY

What you committed

  • Commits per day this week
  • Deadline crammer detection
  • Peak coding hour
  • Progress since week 1
  • Streak and milestones
LAYER 2 — FILE ACTIVITY

What you actually did

  • File open, modify, create events
  • Timestamped activity log
  • Days you worked without committing
  • Real effort vs git history
  • Automatic — just run watch

CLI Commands
Seven commands. Zero setup.

Point gitgrade at any git repository on your machine.

gitgrade scan .
Full overview: total commits, peak coding hour, and milestone check.
gitgrade habits .
How many days did you code this week? Consistent or deadline crammer?
gitgrade progress .
Compare your first week of commits to this week. See your real growth.
gitgrade patterns .
Find your peak coding hour and get advice on your learning schedule.
gitgrade milestones .
Track your streak, total commits, and active coding days.
gitgrade watch .
Watch your project folder and automatically log all file activity with timestamps.
gitgrade compare .
Compare file activity log with git commits. Find days you worked but did not commit.
gitgrade badge .
Generate GitHub profile badges showing your streak, commits and coding style. Paste them into your GitHub profile README.
